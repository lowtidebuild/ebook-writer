# Generate Ebook

Generate a complete ebook from the given topic.

## Usage
```
/generate <topic> [--plugin <domain>] [--language <ko|en>] [--author <name>] [--bilingual]
```

## Examples
```
/generate "Claude Code for Lawyers" --plugin legal --author "이기쁨"
/generate "Introduction to Python Programming" --language en --author "John Doe"
/generate "AI 활용 가이드" --bilingual --author "홍길동"
```

## Execution

1. **Parse arguments**:
   - Extract the topic from $ARGUMENTS
   - If `--plugin <domain>` is specified, verify `.claude/plugins/<domain>/PLUGIN.md` exists
   - If no plugin specified, auto-detect: check if any plugin directory exists in `.claude/plugins/`
   - **Language detection** (in order of priority):
     1. If `--language <code>` is specified, use it
     2. Otherwise, auto-detect from the user's input language:
        - Topic in Korean → `primary_language: ko`
        - Topic in English → `primary_language: en`
   - If `--author <name>` is specified, use as author name
     - If not specified, ask the user for author name (do NOT default to "Generated by Ebook Writer Agent")
   - **Bilingual mode**:
     - If `--bilingual` flag is present → `bilingual: true`
     - Otherwise, ask the user: "번역본도 함께 만들까요? (Do you also need a translated version?)"
     - If yes → `bilingual: true`, set secondary_language automatically (ko↔en)
     - If no → `bilingual: false`, skip translation step, produce single-language output only

2. **Check for existing pipeline**:
   - If `output/pipeline_state.json` exists, offer to resume or restart

3. **Initialize new pipeline**:
   - Create `output/pipeline_state.json` per the schema in CLAUDE.md
   - Set `bilingual` field based on user's choice
   - Ensure all output subdirectories exist

4. **Execute pipeline**:
   - Follow the Step Execution Protocol in CLAUDE.md (Steps 1-8 with Gates)
   - If `bilingual: false`, Step 6 (Translation) is skipped, Step 7 builds one PDF, Step 8 builds single-language viewer
   - Each step updates the state file upon completion
   - At each gate, pause and wait for user input

5. **Completion**:
   - When Gate 2 is approved, display final summary with all deliverable paths
